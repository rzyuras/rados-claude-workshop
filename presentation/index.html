Lecciones

Lo que aprendi usandolo

Que funciona, que no, como sacarle el jugo

Leccion 1

Nunca codear sin plan

Sin plan

"agrega endpoint de favoritos"

→ empieza a codear
→ se da cuenta que falta un consumer
→ reescribe la mitad
→ resultado inconsistente

Con plan

"agrega endpoint de favoritos"

→ investiga codebase
→ identifica 3 repos afectados
→ presenta plan con 4 subtasks
→ yo apruebo
→ ejecuta limpio a la primera

Por eso el orchestrator tiene checkpoint obligatorio. Analyst planea → yo apruebo → recien ahi codean los dev agents.

Leccion 2

Se opinionated

Instruccion vaga → codigo generico. Tu opinion es el input mas valioso.

VAGO

"escribe un controlador para bookings"

OPINIONATED

"controlador Express, Zod validation en boundary, error handling centralizado, patron hexagonal"

Investigar leer, explorar
→
Aprender patrones, trade-offs
→
Opinar decidir approach
→
Skill cristalizar

Una vez cristalizada → Skill. Ya no la repitas nunca mas.

Leccion 3

Hablar en arquitectura, no en codigo

MICRO (no hagas esto)

"escribe una clase BookingConsumer que herede de Karafka::BaseConsumer con un metodo consume que parsee el payload JSON..."

MACRO (hace esto)

"consumer Karafka que procese BOOKING_SCHEDULED, enriquezca con datos del cliente, y emita CHRONOS_BOOKING_SCHEDULED"

Define el que y el por que. El agente rellena el como (ya tiene tus Skills con convenciones).

Leccion 4

CI/CD es tu safety net

"Como confias en codigo que no escribiste?"

Pre-commit

Husky hooks
Lint + format

Pre-flight (/ship)

RuboCop / ESLint
Type-check
Tests (100% coverage)

CI Pipeline

GitHub Actions
Build + test + deploy
Auto-merge si <200 lineas

No necesitas leer cada linea. Necesitas un pipeline en el que confias.

Flags que uso

-c

Retoma ultima sesion (con todo el contexto)

-p "prompt"

Print mode — ejecuta y sale. Para CI/CD o scripts.

claude -p "genera changelog de este PR"

--add-dir PATH

Multi-repo: agrega otro repo al contexto

--dangerously

Flujos para automatizar

Code review automatizado

Reviewer agent que corre en cada PR. Reporta BLOCKER / WARNING / NIT. Ya lo tengo — funciona bien.

Onboarding instantaneo

CLAUDE.md + skills = dev nuevo productivo desde dia 1. Clona el repo y ya tiene las convenciones del equipo.

Migration assistant

Describe el plan de migracion. El agente lo ejecuta archivo por archivo. Ideal para upgrades de framework.

Incident response

Agent que lee logs, diagnostica el error, propone fix. claude -p "diagnostica este error: ..."

Changelog + release notes

claude -p en tu CI para generar changelog automatico de cada release.

Test generation

Apuntar a un archivo y pedirle que genere tests. Con un Skill de testing sigue tus convenciones.
